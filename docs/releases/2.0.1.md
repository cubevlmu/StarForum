# StarForum 2.0.1 Release

This update focuses on long-discussion reading, editor drafts, sticky discussion synchronization, notification compatibility, and local data management. It also resolves several display issues caused by incomplete API responses and differences between Flarum extensions.

## Highlights

- Added multiple manually managed drafts for new discussions and replies.
- Drafts are isolated by forum, account, and target, with legacy draft migration.
- Added back-to-top actions and progressive desktop wheel acceleration.
- Fixed pagination for discussions with large reply histories.
- Improved hot reply ordering and long-list rendering performance.
- Added support for sticky, super-sticky, locked, and unavailable post events.
- Added rendering and navigation support for `newPost` notifications.
- Added an option to keep cached sticky discussions at the top, enabled by default.

## Fixes and Improvements

- Recognize `viewCount`, `views`, and `viewsCount`.
- Recognize both `isSticky` and `isStickiest`.
- Preserve valid cached metadata when remote responses are incomplete.
- Exclude the first post from reply totals.
- Avoid displaying invalid dates, view counts, or reply counts as zero.
- Render empty comments and unknown events as explicit status items.

## Data Management

- Replaced Cache Management with categorized Data Management.
- Show database, settings, logs, image caches, and temporary export usage.
- Restrict Windows storage accounting to StarForum-owned files.
- Allow individual cleanup of cached resources, images, logs, and exports.
- Compact SQLite after cache cleanup to release disk space.

## Interface and Documentation

- Added a project introduction and improved the About page description.
- Update checks now clearly report whether the app is current or an update is available, followed by the release notes.
- Fixed duplicate headers on the account page and added descriptive subtitles.
- Refreshed English and Chinese documentation and localization resources.

# StarForum 2.0.1 更新

本次更新集中完善了长贴文阅读、编辑器草稿、置顶主题同步、通知适配和本地数据管理，并修复了多个由不同 Flarum 扩展和不完整 API 数据引起的显示问题。

## 主要更新

- 编辑器新增多草稿管理，可手动保存、恢复和删除主题或回复草稿。
- 草稿按照论坛、账户和编辑目标隔离，并自动迁移旧版草稿数据。
- 长贴文新增返回顶部按钮，桌面端连续滚轮支持渐进加速。
- 修复大量回复的分页加载，并改进热门回复排序和列表渲染性能。
- 支持普通置顶、超级置顶、锁定及内容不可用等贴文状态。
- 新增 `newPost` 通知类型的内容摘要、图标和页面跳转支持。
- 新增“始终将置顶主题保持在顶部”设置，默认开启。

## 修复与优化

- 兼容 `viewCount`、`views`、`viewsCount` 等浏览量字段。
- 兼容 `isSticky` 与 `isStickiest`，避免超级置顶状态丢失。
- 服务端数据不完整时保留本地有效的标题、日期、浏览量和回复统计。
- 回复数量不再包含首帖，缺失数据不再错误显示为 `0`。
- 空评论和未知状态不再渲染为空白回复。
- 首页与贴文列表会隐藏无效日期、浏览量和回复统计。

## 数据管理

- “缓存管理”升级为“数据管理”。
- 分类显示数据库、设置、日志、图片缓存和临时导出文件占用。
- Windows 不再扫描整个 AppData 或系统临时目录。
- 支持分别清理讨论、回复、用户、标签、通知、图片、日志和导出文件。
- 清理数据库缓存后执行空间整理，实际释放 SQLite 占用。

## 界面与文档

- 关于页面新增项目介绍，并优化应用描述。
- 更新检查会明确提示“当前已是最新版本”或“发现新版本”，同时显示版本日志。
- 修复“我的”页面重复标题，并补充账户页和用户页说明。
- 更新中英文 README 和多语言界面文案。
