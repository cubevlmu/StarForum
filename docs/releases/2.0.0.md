# StarForum 2.0 Test Release

[简体中文](#简体中文) | [English](#english)

## 简体中文

StarForum 2.0 是一次覆盖界面、数据、网络和工程结构的完整重构。这个版本不只是调整外观，而是重新设计了客户端与 Flarum、SQLite 缓存及多平台界面之间的协作方式，为后续稳定迭代建立更清晰的基础。

2.0 当前作为公开测试版本发布，欢迎在实际 Flarum 站点、不同网络环境和移动端/桌面端设备上进行体验和反馈。

### 全新的 FinUI 跨平台界面

- 全面迁移到 FinUI 设计体系，统一页面、导航、按钮、对话框、标签、提示和刷新交互。
- 为手机、平板和桌面端重新设计响应式导航、分栏布局与页面安全区域。
- 重做下拉刷新、上拉加载、骨架屏、空状态和加载反馈，减少布局跳动与无意义提示。
- 改进深色模式、动态强调色以及移动端系统状态栏和底部安全区域表现。
- 更新应用图标、关于页面和整体视觉细节。

### 本地优先的增量同步

- 使用 Drift 与 SQLite 重构本地数据层，讨论、帖子、用户、标签、通知和集合缓存拥有明确的数据边界。
- 服务端数据优先写入本地数据库，页面通过本地投影持续更新，降低网络波动对浏览体验的影响。
- 引入数据指纹、更新时间、有效期和分页集合状态，只同步发生变化或已经过期的数据。
- 首页请求直接包含首帖、作者和标签关系，减少不必要的逐条补取。
- 首帖摘要、用户资料和标签关系支持后台补全与后续同步修复。
- 缓存写入、清理和迁移逻辑集中到数据层，UI 不再直接依赖数据库实现。

### 更清晰的数据与会话架构

- 将大型 Repository 拆分为信息流、详情、写操作、缓存写入和摘要补全等独立职责。
- 将登录会话与用户资料、用户目录、权限和资料修改分离。
- Repository 不再反向查找页面 Controller，登录状态通过独立 Session 状态分发。
- 领域模型逐步改为不可变对象，并区分列表摘要与详情数据，减少可变聚合对象造成的状态错误。
- 登录流程改为验证后提交，避免登录失败时残留不完整的认证状态。

### 网络、缓存与性能改进

- 使用统一的 Flarum API Client 与传输层，移除旧的全局 HttpUtils 请求接口。
- 网络层返回结构化错误，由 UI 决定如何展示提示，修复拦截器重复完成请求的问题。
- 合并相同的安全 GET 请求，同时避免带取消令牌的页面请求相互取消。
- 优化 JSON 解析、批量数据库读写、用户补全、图片解码尺寸和内容缓存占用。
- 为 Repository、网络、解析、SQLite、缓存命中、请求合并、HTML 解析、后台任务和页面首屏增加性能指标。
- 提供数据层 Benchmark，便于在同一设备和网络条件下持续比较性能。

### 讨论、帖子与内容阅读

- 首页、搜索、关注列表、用户主题和文章详情统一使用新的讨论卡片。
- 支持显示讨论标签、普通置顶与超级置顶状态。
- Flarum 状态事件不再被渲染为空回复；已适配置顶与超级置顶事件，未知类型会显示原始事件键。
- 修正回复总数，将首帖与真实回复分开统计。
- 回复支持热度、时间正序和时间倒序排序，并优化小屏幕下的排序控件。
- 点赞状态会即时高亮，关注状态与回复列表可以在操作后同步更新。
- 改进长 HTML、代码块、图片、SVG、相对地址和图片预览的渲染。
- 自动移除脚本、样式和其他非正文节点，避免代码高亮脚本被错误显示为帖子内容。

### 编辑器、媒体与用户功能

- 提供新的富文本编辑体验，支持发帖、回复、链接、代码和图片内容。
- 修复移动端标题、正文焦点和标签选择交互，支持主标签与普通标签。
- 增加媒体资源管理与上传流程。
- 重构用户资料页，将资料、回复、主题和徽章拆分为独立状态模块。
- 改进头像选择、裁剪、上传以及 Bio、昵称等资料修改流程。
- 修复用户组筛选与管理员、版主等组别显示。
- 改进通知加载、全部已读和清理操作。

### 设置、诊断与本地化

- 支持英语、简体中文、繁体中文（台湾）、日语、韩语和越南语界面。
- 修复语言切换后名称仍显示旧语言的问题。
- 将手动检查更新移动到“通用”设置，并保留可配置的启动检查。
- 增加更完整的缓存管理和分类清理能力。
- 开发者页面可以查看客户端、系统和存储信息，并导出 SQLite 数据库与本地设置用于诊断。
- Release 构建可在“关于”页面连续点击版本号六次进入开发者页面。

### 升级说明

- 2.0 使用了新的本地数据库结构。从旧版本升级时，应用会清除并重建不兼容的数据库缓存。
- 其他本地设置不会因这次数据库迁移被清除。
- 首次启动后需要重新同步论坛数据；同步时间取决于站点响应速度、数据量和网络环境。
- 这是测试版本。建议在反馈问题时附上客户端版本、系统平台、Flarum 版本、启用的扩展以及开发者页面导出的诊断信息。

感谢所有参与测试、提交日志和提供兼容性反馈的用户。

## English

StarForum 2.0 is a full redesign of the interface, data layer, networking stack, and project structure. It goes beyond a visual refresh: the way the client coordinates Flarum APIs, SQLite caches, and responsive multi-platform screens has been rebuilt to provide a cleaner foundation for future releases.

Version 2.0 is currently distributed as a public test release. Feedback from real Flarum installations, different network conditions, and both mobile and desktop devices is welcome.

### A New Cross-Platform FinUI Experience

- Migrated the application to the FinUI design system, including navigation, pages, buttons, dialogs, tags, notifications, and refresh interactions.
- Redesigned responsive navigation, split views, and safe-area handling for phones, tablets, and desktop windows.
- Reworked pull-to-refresh, load-more indicators, skeletons, empty states, and loading feedback to reduce layout shifts and persistent status UI.
- Improved dark mode, dynamic accent colors, and mobile status/navigation bar behavior.
- Refreshed application icons, the About screen, and visual details throughout the client.

### Offline-First Incremental Synchronization

- Rebuilt local persistence with Drift and SQLite, with explicit boundaries for discussions, posts, users, tags, notifications, and cached collections.
- Server responses are persisted locally first and screens update from database projections, making browsing less sensitive to unstable networks.
- Added fingerprints, update timestamps, freshness windows, and paged collection state so unchanged data does not need to be downloaded repeatedly.
- Discussion feeds request first posts, authors, and tag relationships together to avoid per-item follow-up requests.
- First-post excerpts, incomplete user records, and discussion tags can be hydrated in the background and repaired by later synchronization.
- Cache writes, cleanup, and migrations are owned by the data layer instead of presentation code.

### Cleaner Data and Session Architecture

- Split oversized repositories into focused feed, detail, mutation, cache-writing, and excerpt-hydration components.
- Separated login sessions from user profiles, directories, permissions, and profile mutations.
- Removed repository dependencies on page controllers; session changes now flow through dedicated session state.
- Moved domain data toward immutable models and separated lightweight list summaries from detailed discussion data.
- Changed login to a validate-then-commit transaction so failed authentication cannot leave a partial local session.

### Networking, Caching, and Performance

- Standardized requests on the Flarum API client and transport layer, replacing the legacy global HttpUtils request surface.
- Network code now produces structured failures while presentation code decides how to notify the user.
- Added safe request coalescing without allowing one page's cancellation token to cancel another page's request.
- Optimized JSON parsing, batched database operations, user hydration, image decode sizing, and bounded content caches.
- Added metrics for repositories, networking, parsing, SQLite, cache hits, request coalescing, HTML parsing, background work, and first-interactive timing.
- Added a repeatable data-layer benchmark workflow for performance comparisons on the same device and network.

### Discussions, Posts, and Reading

- Unified discussion cards across the home feed, search, subscriptions, user topics, and discussion details.
- Added discussion tags, pinned markers, and super-sticky event support.
- Flarum status events are no longer rendered as empty replies. Sticky and super-sticky events are supported, while unknown events expose their original event key.
- Corrected reply totals so the first post is not counted as a reply.
- Added hot, oldest-first, and newest-first reply ordering with a compact control for smaller screens.
- Like state now updates visually, while follow state and reply lists stay synchronized after actions.
- Improved rendering for long HTML, code blocks, images, SVG resources, relative URLs, and full-screen image previews.
- Scripts, styles, and other non-content nodes are removed before rendering so syntax-highlighting scripts cannot appear as post text.

### Editor, Media, and User Features

- Added a redesigned rich-content editor for discussions and replies, including links, code, and images.
- Fixed mobile title/body focus transitions and tag selection, including both primary and secondary tags.
- Added media asset management and upload workflows.
- Reorganized user profiles into focused profile, reply, topic, and badge states.
- Improved avatar selection, cropping, upload, biography editing, and nickname updates.
- Fixed user-group filtering for administrators, moderators, and other groups.
- Improved notification loading, mark-all-read, and clearing actions.

### Settings, Diagnostics, and Localization

- Added interfaces for English, Simplified Chinese, Traditional Chinese (Taiwan), Japanese, Korean, and Vietnamese.
- Fixed language labels that previously remained in the old locale after switching languages.
- Moved manual update checks to General settings while retaining configurable startup checks.
- Expanded cache inspection and category-based cleanup controls.
- The developer page now reports application, system, and storage information and can export the SQLite database and local settings for diagnostics.
- Release builds can unlock the developer page by tapping the version label six times on the About screen.

### Upgrade Notes

- StarForum 2.0 uses a new local database schema. Upgrading from an older version clears and rebuilds incompatible database caches.
- Other local preferences are preserved by this database migration.
- Forum data must synchronize again after the first launch. The time required depends on server latency, data volume, and network conditions.
- This is a test release. When reporting an issue, include the client version, platform, Flarum version, enabled extensions, and diagnostics exported from the developer page.

Thank you to everyone testing the release, sharing logs, and reporting compatibility issues.
